# 项目适配指南

本指南用于准备项目级指令候选。它不编造仓库事实，也不授权写入。确认、写入、回读和验证遵循[安装指南](INSTALL.zh-CN.md)的统一流程。

## 作用范围与来源

收集事实前，先确认目标项目和请求范围。只读取确定约束所需材料：

- 适用的 `AGENTS.md` 文件及其可见父级/子级层级；
- 仓库 README 和贡献说明；
- 构建配置、CI、包或依赖清单以及已记录命令；
- 相关源码目录和受保护区域；
- Git 状态，以及定义验收、兼容性、发布或部署约束的项目文档。

可见指令层级不等于某个 Agent 产品的加载顺序已经得到证明，除非该产品行为实际可验证。记录文件的表面作用范围；不能解决的冲突必须升级说明，不得猜测优先级。

## 分类已知信息

| 类别 | 候选中的允许处理 |
| --- | --- |
| 已验证项目事实 | 可写入，并保留其证据或来源。例如已记录命令、目录职责、工具链约束和兼容性承诺。 |
| 用户偏好 | 明确标为偏好，或留给适用全局规则；不得写成仓库要求。 |
| 未知信息 | 留空、标记待确认，或只提出最少必要问题；不得根据惯例推断。 |

不得编造命令、路径、目录职责、部署流程、配置位置、API 或 Agent 配置位置。不得把私有路径、凭据、会话资料或机器元数据复制进项目候选。

## 选择 overlay 或 standalone

只有确认兼容全局规则已经适用时，才选用 overlay。overlay 仅补充已验证项目事实和项目约束；不得重复、替代或削弱全局安全、敏感信息、删除、明确确认或外部操作边界。

不存在兼容全局规则、无法确认其适用性，或项目需要自包含最低安全边界时，选用 standalone。独立模板包含必要边界，且必须保留。

| 判断 | 模板 |
| --- | --- |
| 兼容全局规则已经适用 | [英文 overlay](../templates/project/AGENTS.overlay.md) 或[中文 overlay](../templates/project/AGENTS.overlay.zh-CN.md) |
| 没有兼容全局规则，或需要自包含规则 | [英文 standalone](../templates/project/AGENTS.standalone.md) 或[中文 standalone](../templates/project/AGENTS.standalone.zh-CN.md) |
| 适用性仍未知 | 保持仅候选；选择写入目标前，只提出最少必要问题。 |

当前用户偏好或授权策略影响候选时，参阅[裁剪指南](CUSTOMIZE.zh-CN.md)。

## 准备候选和 diff

候选必须标明：

- 目标文件及其已发现指令关系；
- 选用 overlay 或 standalone 的模式和理由；
- 项目权威来源，以及每项项目事实的证据；
- 保留、新增、修改和删除的内容；
- 未验证的空白或待确认字段；
- 验收检查、验证方法和回滚方法。

目标已存在时，展示可审阅 diff；目标不存在时，展示完整的新文件候选。准备这些材料时，不得写入、合并、替换、创建备份或修改项目。

## 处理冲突

可见指令冲突时，说明受影响范围、相互冲突的要求和实际影响。overlay 模式必须保留全局安全边界。若无法依据可用指令和已验证产品行为化解冲突，先请求方向再继续。

## 进入安装流程

用户审阅候选后，遵循[安装指南](INSTALL.zh-CN.md)：展示完整写入影响，等待明确确认，然后写入或合并并回读实际结果。已验证的项目构建或测试不构成提交、推送、部署或发布授权。
