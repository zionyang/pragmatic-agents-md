# 安装指南

本指南说明如何准备可复制模板，并且只在获得明确确认后，将其写入 Agent 或项目环境。它不是一键安装器，也不为任何 Agent 产品预设配置路径、文件名、加载行为或命令。

请先通过[仓库 README](../README.zh-CN.md)选择语言，通过[模板索引](../templates/README.md)选择起点；裁剪和项目事实分别见下方链接的其他指南。模板正文仍是规则本身的权威来源。

## 支持证据

只有读取了 Agent 的实际配置入口、现有配置或官方文档，并且这些证据确认了相关行为时，才能说该 Agent 产品支持某次具体安装。不得根据产品名称、其他操作系统或相似 Agent 推断路径。

对于未知 Agent，只能提供候选，并说明尚未验证的部分。不得声称文件会被自动加载。人类用户仍可把已审阅文本复制到自己确认合适的 Agent 入口，但本指南不会替其作出该选择。

## 统一流程

所有安装模式都使用以下顺序：

1. 读取仓库说明和适用模板。
2. 只读发现目标环境与既有指令。
3. 选择模式和源模板。
4. 仅使用已验证信息生成候选。
5. 展示目标、理由、diff、风险、验证方式和回滚方式。
6. 等待用户对拟议写入或合并作出明确确认。
7. 仅在确认后写入或合并。
8. 回读实际结果，并验证其内容、作用范围以及实际能够验证的加载行为。

仅生成候选的分支在第 5 步停止；不得创建文件、备份、安装、发起网络请求、提交、推送、部署或发布。

## 只读发现

选择模式前，只检查确定目标和约束所需的信息：

- 用户请求的作用范围：全局、项目根目录、指定项目子目录或仅生成候选；
- 请求特定产品目标时，Agent 的现有配置和已验证官方文档；
- 现有全局和项目指令文件，以及其可见层级与作用范围；
- 目标项目的可用指令、README、贡献说明、构建配置、CI 和 Git 状态；
- 用户是否已经有兼容的全局规则。

检查这些来源时，不得输出或复制凭据、token、密码、cookie、环境变量值、私有路径、机器元数据或会话资料。

## 选择模式

| 模式 | 适用条件 | 候选来源 | 边界 |
| --- | --- | --- | --- |
| 全局 | 用户需要跨项目稳定行为，且实际全局入口能够验证。 | 用户语言对应的[全局模板](../templates/global/README.md)。 | 与现有全局规则比较，优先生成可审阅的合并候选。 |
| 项目 overlay | 兼容的全局规则已经适用于该项目。 | [英文 overlay](../templates/project/AGENTS.overlay.md) 或[中文 overlay](../templates/project/AGENTS.overlay.zh-CN.md)。 | 只新增已验证项目事实；不得重复或削弱全局安全边界。 |
| 项目 standalone | 没有兼容全局规则，或项目需要自包含的最低安全边界。 | [英文 standalone](../templates/project/AGENTS.standalone.md) 或[中文 standalone](../templates/project/AGENTS.standalone.zh-CN.md)。 | 保留独立模板中的最低安全边界。 |
| 仅生成候选 | 用户只要草案、审阅或比较，或尚未授权写入。 | 适用模板和已验证证据。 | 展示候选和 diff 后停止。 |
| 未知 Agent | 无法验证 Agent 的入口或加载行为。 | 适用模板的纯文本候选。 | 不猜测目标，也不承诺自动安装。 |

选择项目 overlay 或 standalone 前，先阅读[项目适配指南](PROJECT.zh-CN.md)。候选需要区分稳定安全边界、偏好和授权策略时，阅读[裁剪指南](CUSTOMIZE.zh-CN.md)。

## 准备候选

只能使用已验证项目事实、现有适用规则和用户明确指示。未知信息必须保留为空或标记为待确认，不得猜测。目标已存在时，展示与现有内容的比较；目标不存在时，展示完整的新文件候选，并明确它是新增文件。

请求写入确认前，必须说明：

- 已发现的确切目标；如果尚未验证目标，则明确说明；
- 选用模式和源模板；
- 与现有全局、父级或子级指令的关系；
- 保留、新增、修改或删除的规则和事实；
- 验证方法，以及实际加载能否验证的限制；
- 回滚方法，以及无法可靠回滚的限制。

## 写入与合并规则

用户未对该次操作明确确认前，不得写入、覆盖、合并、创建备份、安装依赖、联网、提交、推送、部署或发布。

获得确认的写入授权后，优先与现有规则合并。仅在目标不存在时创建新文件；仅在用户明确选择替换时覆盖现有目标。需要备份时，创建前先说明其位置、敏感性和保留策略。

构建、测试、预览、验收检查或 diff 审阅成功，不构成提交、推送、部署、发布或其他独立外部操作的授权。

## 回读验证

确认后的写入或合并完成后：

1. 回读实际目标文件。
2. 与已确认的候选比较。
3. 确认没有把密钥、私有路径、机器元数据或会话资料写入目标。
4. 对项目文件，检查可见父级/子级指令关系，并报告未解决冲突。
5. 只有存在实际支持的方法时才验证加载；若无法验证，明确说明文件已写入但加载仍未验证。
