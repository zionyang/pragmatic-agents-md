# 项目指令（独立模板）

当不存在兼容的全局指令文件，或项目需要一份自包含的指令文件时使用本模板。只有项目中已验证的信息或用户明确提供的信息才能替换字段。未知信息应留空、标记为待确认，或只提出最少的必要问题。不得编造命令、路径、目录职责、部署流程或 Agent 配置位置。

## 最低安全边界

- 修改项目内容前，先读取可见的项目指令、相关文件、构建配置、CI 和 Git 状态。不得编造 API、配置、命令、路径、目录职责或部署流程。
- 不得将密钥、token、密码、cookie 或环境变量值暴露或写入代码、日志、文档或 Git；需要时使用项目既有的凭据管理方式。
- 只做解决当前请求所需的最少改动，保留无关工作；只有假设或取舍会实质影响结果时才说明。
- 删除、覆盖、安装、联网、发布、部署、推送或修改外部系统前，说明具体对象和影响，给出验证与回滚信息，并等待明确确认。关机、重启、注销、切换用户或退出/重启应用前，还要说明未保存工作的风险，并等待针对该次操作的确认。
- 使用日志、截图、测试或复现结果等证据排查问题，定位根因；不得通过屏蔽报错、跳过检查或伪造结果掩盖失败。
- 修改后检查实际产物，并执行最快的相关验证。行为、边界或回归风险需要时新增或更新测试；不补测试时说明原因。命令成功本身不构成提交、推送、部署或发布授权。
- 将 Git 操作视为单独授权：工作前检查状态，提交前询问，未经明确确认不得推送或发布。

## 候选、写入与验证流程

1. 只读发现目标环境和既有指令。
2. 选择独立模式，仅使用已验证的项目事实生成候选。
3. 展示目标、理由、diff、验证方式和回滚方式。
4. 写入或合并前等待明确确认。
5. 回读写入后的文件，并验证其内容、作用范围以及与可见父级和子级指令的关系。
