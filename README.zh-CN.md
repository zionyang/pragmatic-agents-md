# pragmatic-agents-md

面向人类用户和 AI coding Agent 的可移植 `AGENTS.md` 模板与 Markdown 指南。

[阅读英文版本](README.md)。

## 本仓库提供什么

- 中英文全局 `AGENTS.md` 行为模板。
- 项目级叠加版与独立版模板。
- 用于发现 Agent 实际配置、裁剪候选内容、取得确认、写入并验证结果的操作指南。
- 对规则设计意图、边界与迁移选择的解释。

本仓库不是一键安装器、私有配置备份，也不授权 Agent 在未获明确确认前修改已有项目或 Agent 配置。

## 当前状态

本仓库已完成阶段 3：模板设计与本地化。阶段 4 的安装、裁剪、项目级适配与规则解释操作指南尚未开始。

- [AGENTS.md](AGENTS.md) 是供处理本仓库的 Agent 遵守的 bootstrap 指令文件。
- [全局模板](templates/global/README.md) 已提供英文和简体中文版本。
- [项目模板](templates/project/README.md) 已提供中英文 overlay 与 standalone 版本。
- [guides/](guides/README.md) 仍只是阶段 4 操作指南的目录级占位入口。

模板文件可以作为可复制的起点。不得把目录 README 复制为 `AGENTS.md` 配置，也不得把尚未提供的阶段 4 指南当作安装流程。

## 预期使用方式

可选择以下路径：

1. 直接复制已完成的模板，仅按自身环境调整适用部分。
2. 让 Agent 遵守本仓库的 bootstrap 流程，先只读检查现有规则与目标项目，再生成可审阅的候选内容；未获确认前不得改动任何内容。

现已提供以下模板类别：

- 全局模板：跨项目稳定生效的协作行为。
- 项目叠加模板：面向已有全局规则的用户，仅补充已验证的项目事实与约束。
- 项目独立模板：面向没有全局规则的用户，包含项目规则和必要的最低安全边界。

## Agent 启动提示词

将仓库克隆或交给 Agent 后，可使用以下提示词：

> 请先阅读仓库根目录的 `AGENTS.md`，再阅读与我语言一致的 README。说明当前可用的模板或指南状态；如需处理目标环境，先进行只读检查，且只有所需源材料实际存在时才生成候选。任何写入、覆盖、安装、联网、提交、推送、部署或发布前，先展示目标、理由、diff、验证方式和回滚方式，并等待我的明确确认。

对于未知 Agent 产品，Agent 必须发现其已存在配置或已验证的官方文档；不得猜测路径，也不得声称会自动加载。

## 安全与支持边界

Agent 应遵循以下流程：

```text
读取仓库说明
  -> 只读发现
  -> 选择模式与模板
  -> 生成候选
  -> 展示 diff、风险、验证与回滚
  -> 等待明确确认
  -> 写入或合并
  -> 回读并验证
```

该流程不授权写入、覆盖、安装、联网、提交、推送、部署、发布或其他外部变更。对于未知 Agent 产品，也不承诺固定配置路径或自动加载。
