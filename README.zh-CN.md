# pragmatic-agents-md

给人和 AI coding Agent 用的 `AGENTS.md` 模板与使用说明。

[阅读英文版本](README.md)。

## 怎么用

- 直接复制模板，再按自己的环境调整。
- 也可以让 Agent 读取本项目，先检查你现有的规则和目标项目，再为你准备一份可审阅的修改方案。
- 没有得到你的确认，Agent 不会直接修改配置。

## 模板怎么分

- **全局模板**：适合放跨项目都要遵守的习惯，例如用户语言、输入习惯、回答风格，以及处理文件、联网和 Git 操作前的确认边界。
- **项目叠加模板**：适合已经有兼容全局规则、只需要补充某个项目自身信息的情况，例如目录职责、构建和测试命令、发布限制或不能修改的区域。
- **项目独立模板**：适合没有合适的全局规则，或希望项目自身带有必要安全边界的情况。

## 本仓库提供什么

- 中英文全局 `AGENTS.md` 行为模板。
- 项目级叠加版与独立版模板。
- 用于发现 Agent 实际配置、裁剪候选内容、取得确认、写入并验证结果的操作指南。
- 对规则设计意图、边界与迁移选择的解释。

本仓库不是一键安装器、私有配置备份，也不授权 Agent 在未获明确确认前修改已有项目或 Agent 配置。

## 已提供的材料

- [AGENTS.md](AGENTS.md) 是供处理本仓库的 Agent 遵守的 bootstrap 指令文件。
- [全局模板](templates/global/README.zh-CN.md) 已提供英文和简体中文版本。
- [项目模板](templates/project/README.zh-CN.md) 已提供中英文 overlay 与 standalone 版本。
- [指南](guides/README.zh-CN.md) 已提供中英文安装、裁剪和项目级适配流程。
- [规则说明](GUIDE.zh-CN.md)与[英文规则说明](GUIDE.md)解释 23 条全局规则及其迁移边界。

模板文件可以作为可复制的起点。不得把目录 README 复制为 `AGENTS.md` 配置。指南描述的是带确认门的流程，本身不授权任何环境变更。

## Agent 启动提示词

将仓库克隆或交给 Agent 后，可使用以下提示词：

> 请先阅读仓库根目录的 `AGENTS.md`，再阅读与我语言一致的 README。说明当前可用的模板或指南状态；如需处理目标环境，先进行只读检查，且只有所需源材料实际存在时才生成候选。任何写入、覆盖、安装、联网、提交、推送、部署或发布前，先展示目标、理由、diff、验证方式和回滚方式，并等待我的明确确认。

对于未知 Agent 产品，Agent 必须发现其已存在配置或已验证的官方文档；不得猜测路径，也不得声称会自动加载。

## 安全与支持边界

Agent 应遵循以下流程：

```text
读取仓库说明
  -> 只读发现
  -> 选择模式与模板
  -> 生成候选
  -> 展示 diff、风险、验证与回滚
  -> 等待明确确认
  -> 写入或合并
  -> 回读并验证
```

该流程不授权写入、覆盖、安装、联网、提交、推送、部署、发布或其他外部变更。对于未知 Agent 产品，也不承诺固定配置路径或自动加载。
